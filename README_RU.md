# 🎵 Ter-Music-Rust - Терминальный музыкальный плеер 🎵

</div>

Простой и практичный терминальный музыкальный плеер на Rust с поддержкой локального/сетевого поиска и загрузки треков, автоматической загрузки и отображения текстов песен, просмотра комментариев, переключения языка и темы. Поддерживаются Windows, Linux и MacOS.

## ✨ Функции

### 🎵 Воспроизведение аудио
- **Поддержка 10 аудиоформатов**: MP3, WAV, FLAC, OGG, OGA, Opus, M4A, AAC, AIFF, APE
- **Управление воспроизведением**: воспроизведение/пауза/стоп, предыдущий/следующий трек
- **Перемотка**: быстрая перемотка на 5с / 10с
- **Перемотка по прогресс-бару**: нажмите на прогресс-бар для точного перехода
- **Управление громкостью**: регулировка в реальном времени от 0-100, нажмите на шкалу громкости для установки

### 🔄 5 режимов воспроизведения
| Клавиша | Режим | Описание |
|------|------|------|
| `1` | Одиночное воспроизведение | Останов после текущего трека |
| `2` | Повтор одного трека | Повтор текущего трека |
| `3` | Последовательное воспроизведение | Воспроизведение по порядку, останов в конце |
| `4` | Повтор списка | Повтор всего плейлиста |
| `5` | Случайное воспроизведение | Случайный выбор треков |

### 📜 Система текстов песен
- **Загрузка локальных текстов**: автоматический поиск соответствующих `.lrc` файлов
- **Определение кодировки текстов**: автоопределение UTF-8 / GBK
- **Автоматическая загрузка из сети**: асинхронная фоновая загрузка при отсутствии локальных текстов
- **Прокрутка с подсветкой**: текущая строка выделяется `►`, автопрокрутка с центрированием
- **Переход по тексту**: перетащите область текстов или используйте колёсико мыши для перехода по временным меткам

### 🔍 Поиск
- **Локальный поиск**: нажмите `s` для поиска песен в текущем музыкальном каталоге
- **Онлайн-поиск**: нажмите `n` для поиска онлайн-песен по ключевому слову
- **Агрегированный поиск**: нажмите `j` для входа. Поиск агрегированных песен по совпадению ключевых слов.
- **Поиск плейлистов**: нажмите `p` для входа. Поиск онлайн-плейлистов по совпадению ключевых слов.
- **Постраничный просмотр**: `PgUp` / `PgDn` для большего количества результатов
- **Онлайн-загрузка**: нажмите `Enter` на выбранном онлайн-результате для загрузки в текущий музыкальный каталог (с отображением прогресса)

### 🤖 Информация о песне
- **Умный запрос**: нажмите `i` для запроса подробной информации о песне, поддерживает любой API, совместимый с OpenAI
- **Потоковый вывод**: результаты отображаются посимвольно, не нужно ждать полной генерации
- **Богатая информация**: охватывает 13 категорий, включая подробности об артисте, авторство, список треков альбома, творческий фон, значение текста, музыкальный стиль, анекдоты и многое другое
- **Многоязычная поддержка**: язык ответа следует настройке языка UI (Упр.кит./Трад.кит./Англ./Яп./Кор.)
- **Пользовательский API**: нажмите `k` для настройки базового URL API, API Key и имени модели за 3 шага — поддерживает DeepSeek, OpenRouter, AIHubMix и любую совместимую с OpenAI конечную точку
- **Бесплатный резерв**: автоматически использует бесплатную модель OpenRouter (minimax/minimax-m2.5:free), если API Key не настроен

### ⭐ Избранное
- **Добавить/удалить из избранного**: нажмите `f` для переключения статуса избранного текущего трека
- **Список избранного**: нажмите `v` для просмотра избранного (с маркером `★`)
- **Воспроизведение из другого каталога**: автоматическое переключение каталога, когда избранное находится в другом каталоге
- **Удалить из избранного**: нажмите `d` в списке избранного

### 💬 Комментарии
- **Комментарии к песне**: нажмите `c` для просмотра комментариев к текущей песне
- **Подробности комментария**: нажмите `Enter` для переключения между списком/подробным просмотром (полный текст в подробном просмотре)
- **Отображение ответов**: показывает исходный текст комментария-ответа, никнейм и время
- **Постраничный просмотр комментариев**: `PgUp` / `PgDn`, 20 комментариев на страницу
- **Фоновая загрузка**: комментарии загружаются в фоновых потоках без блокировки UI

### 📂 Управление каталогами
- **Выбор музыкального каталога**: нажмите `o` для открытия диалога выбора папки (воспроизведение начинается автоматически после первого успешного открытия)
- **История каталогов**: нажмите `m` для просмотра и быстрого переключения каталогов
- **Маркер текущего каталога**: `▶` указывает на текущий активный каталог
- **Удалить запись истории**: нажмите `d` в просмотре истории

### 🖱️ Взаимодействие с мышью
- **Клик по плейлисту**: нажмите для воспроизведения песни
- **Клик по прогресс-бару**: переход в указанную позицию
- **Клик по шкале громкости**: регулировка громкости
- **Перетаскивание текстов**: перетащите левой кнопкой для перехода по временным меткам
- **Колёсико мыши на текстах**: прокрутка вверх/вниз для перехода к предыдущей/следующей строке текста
- **Клик по результату поиска**: локальный поиск — воспроизведение, онлайн-поиск — загрузка
- **Клик по комментарию**: открыть подробности

### 📊 Визуализация волновой формы
- Динамические полосы волновой формы на основе реальной RMS громкости при воспроизведении
- EMA-сглаживание для более мягкой визуализации
- Заморозка волновой формы при паузе

### ⚙️ Постоянная конфигурация
Конфигурация хранится в `USERPROFILE/ter-music-rust/config.json` в каталоге программы и автоматически сохраняется/восстанавливается:

| Параметр | Описание |
|--------|------|
| `music_directory` | Последний открытый музыкальный каталог |
| `play_mode` | Режим воспроизведения |
| `current_index` | Индекс последней воспроизведённой песни (возобновление воспроизведения) |
| `volume` | Громкость (0-100) |
| `favorites` | Список избранного |
| `dir_history` | История каталогов |
| `api_key` | API Key (для запроса информации о песне, обратная совместимость с `deepseek_api_key`) |
| `api_base_url` | Базовый URL API (по умолчанию: `https://api.deepseek.com/`) |
| `api_model` | Имя AI модели (по умолчанию: `deepseek-v4-flash`) |
| `github_token` | GitHub Token (используется для отправки обсуждений о песнях; оставьте пустым для использования токена по умолчанию) |
| `theme` | Имя темы |
| `language` | Язык UI (`sc` / `tc` / `en` / `ja` / `ko` / `ru` / `fr` / `de` / `es` / `it` / `pt`) |

**Триггеры автосохранения**: смена трека, смена темы, смена языка, изменение избранного, каждые 30 секунд и при выходе (включая Ctrl+C)

## 🚀 Быстрый старт

### 1. Установите Rust

```powershell
# Способ 1: winget (рекомендуется)
winget install Rustlang.Rustup

# Способ 2: официальный установщик
# Посетите https://rustup.rs/ и установите
```

Проверка установки:

```powershell
rustc --version
cargo --version
```

### 2. Сборка проекта

```powershell
cd <каталог-проекта>

# Способ 1: скрипт сборки (рекомендуется)
build-win.bat

# Способ 2: Cargo
cargo build --release
```

### 3. Запуск

```powershell
# Способ 1: cargo run
cargo run --release

# Способ 2: прямой запуск исполняемого файла
.\target\release\ter-music-rust.exe

# Способ 3: указание музыкального каталога
.\target\release\ter-music-rust.exe -o d:\Music
cargo run --release -- -o d:\Music
```

**Приоритет загрузки каталога**: командная строка `-o` > файл конфигурации > диалог выбора папки

## 📦 Установка и сборка

### Системные требования

- **ОС**: Windows 10/11
- **Rust**: 1.70+
- **Терминал**: Windows Terminal (рекомендуется) / CMD / PowerShell
- **Размер окна**: рекомендуется 80×25 или больше

### Вариант 1: Инструментальная цепочка MSVC (лучшая совместимость, больший размер)

```powershell
# 1. Установите Rust
winget install Rustlang.Rustup

# 2. Установите инструменты сборки
winget install Microsoft.VisualStudio.2022.BuildTools
# Запустите установщик -> выберите «Разработка классических приложений на C++» -> установите

# 3. Перезапустите терминал и соберите
cargo build --release
```

### Вариант 2: Инструментальная цепочка GNU (рекомендуется, легковесная ~300 МБ)

```powershell
# 1. Установите Rust
winget install Rustlang.Rustup

# 2. Установите MSYS2
winget install MSYS2.MSYS2
# В терминале MSYS2 выполните:
pacman -Syu
pacman -S mingw-w64-x86_64-toolchain

# 3. Добавьте PATH (PowerShell от имени администратора)
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\msys64\mingw64\bin", "Machine")

# 4. Переключите инструментальную цепочку и соберите
rustup default stable-x86_64-pc-windows-gnu
cargo build --release
```

> Программы, собранные с инструментальной цепочкой GNU, могут требовать в каталоге исполняемого файла следующие DLL:
> `libgcc_s_seh-1.dll`, `libstdc++-6.dll`, `libwinpthread-1.dll`

### Вариант 3: Кросс-компиляция Linux на Windows

Используйте `cargo-zigbuild` + `zig` в качестве компоновщика. Установка Linux VM/системы не требуется.

```powershell
# 1. Установите zig (выберите один вариант)
# A: через pip (рекомендуется)
pip install ziglang

# B: через MSYS2
pacman -S mingw-w64-x86_64-zig

# C: ручная загрузка
# Посетите https://ziglang.org/download/, распакуйте и добавьте в PATH

# 2. Установите cargo-zigbuild
cargo install cargo-zigbuild

# 3. Добавьте целевую платформу Linux
rustup target add x86_64-unknown-linux-gnu

# 4. Подготовьте Linux sysroot (заголовки/библиотеки ALSA)
# Проект уже включает linux-sysroot/
# При ручной подготовке скопируйте из Debian/Ubuntu:
#   /usr/include/alsa/ -> linux-sysroot/usr/include/alsa/
#   /usr/lib/x86_64-linux-gnu/libasound.so* -> linux-sysroot/usr/lib/x86_64-linux-gnu/

# 5. Сборка
build-linux.bat

# Или выполните вручную:
cargo zigbuild --release --target x86_64-unknown-linux-gnu.2.34
```

**Результат**: `target/x86_64-unknown-linux-gnu/release/ter-music-rust`

**Развёртывание на Linux**:

```bash
# 1. Скопируйте на хост Linux
scp ter-music-rust user@linux-host:~/

# 2. Сделайте исполняемым
chmod +x ter-music-rust

# 3. Установите ALSA runtime
sudo apt install libasound2

# 4. Запустите
./ter-music-rust -o /path/to/music
```

> `build-linux.bat` автоматически настраивает `PKG_CONFIG_PATH`, `PKG_CONFIG_ALLOW_CROSS`, `RUSTFLAGS` и т.д.
> В целевой платформе `x86_64-unknown-linux-gnu.2.34` `.2.34` указывает минимальную версию glibc для лучшей совместимости со старыми системами Linux.

### Упаковка Linux (DEB / RPM)

Если вы собираете/упаковываете на Linux, используйте:

```bash
# 1) RPM
./build-rpm.sh

# Генерация debuginfo RPM (опционально)
./build-rpm.sh --with-debuginfo

# 2) DEB
./build-deb.sh

# Генерация DEB с отладочными символами (опционально)
./build-deb.sh --with-debuginfo

# Генерация пакета исходного кода, совместимого с dpkg-source (.dsc/.orig.tar/.debian.tar)
./build-deb.sh --with-source

# Генерация debuginfo + пакета исходного кода
./build-deb.sh --with-debuginfo --with-source
```

Каталоги вывода по умолчанию:
- `dist/rpm/`: RPM / SRPM
- `dist/deb/`: DEB / пакеты исходного кода

> Скрипты читают `name` и `version` из `Cargo.toml` для автоматического именования файлов пакетов.

### Вариант 4: Кросс-компиляция MacOS на Windows

Используйте `cargo-zigbuild` + `zig` + MacOS SDK. Аудио на MacOS использует CoreAudio и требует заголовков SDK.

**Предварительные требования:**

```powershell
# 1. Установите zig (как для кросс-компиляции Linux)
pip install ziglang

# 2. Установите cargo-zigbuild
cargo install cargo-zigbuild

# 3. Установите LLVM/Clang (предоставляет libclang.dll для bindgen)
# A: через MSYS2
pacman -S mingw-w64-x86_64-clang

# B: официальный LLVM
winget install LLVM.LLVM

# 4. Добавьте целевые платформы MacOS
rustup target add x86_64-apple-darwin aarch64-apple-darwin
```

**Подготовка MacOS SDK:**

Распакуйте `MacOSX13.3.sdk.tar.xz` в `macos-sysroot`.
Проект уже включает `macos-sysroot/` (загружено из [macosx-sdks](https://github.com/joseluisq/macosx-sdks)).

Для повторной загрузки:

```powershell
# A: Загрузите предупакованный SDK с GitHub (рекомендуется, ~56 МБ)
# Зеркало: https://ghfast.top/https://github.com/joseluisq/macosx-sdks/releases/download/13.3/MacOSX13.3.sdk.tar.xz
curl -L -o MacOSX13.3.sdk.tar.xz https://github.com/joseluisq/macosx-sdks/releases/download/13.3/MacOSX13.3.sdk.tar.xz
mkdir macos-sysroot
tar -xf MacOSX13.3.sdk.tar.xz -C macos-sysroot --strip-components=1
del MacOSX13.3.sdk.tar.xz

# B: Скопируйте с системы MacOS
scp -r mac:/Library/Developer/CommandLineTools/SDKs/MacOSX.sdk ./macos-sysroot
```

> Источник SDK: https://github.com/joseluisq/macosx-sdks
> Включает заголовки для CoreAudio, AudioToolbox, AudioUnit, CoreMIDI, OpenAL, IOKit и т.д.

**Сборка:**

```powershell
# Используйте скрипт сборки (автоматически устанавливает все переменные окружения)
build-mac.bat

# Или вручную:
$env:LIBCLANG_PATH = "C:\msys64\mingw64\bin"      # Каталог с libclang.dll
$env:COREAUDIO_SDK_PATH = "./macos-sysroot"         # Путь к MacOS SDK (прямые слеши)
$env:SDKROOT = "./macos-sysroot"                    # Необходимо компоновщику zig для поиска системных библиотек
$FW = "./macos-sysroot/System/Library/Frameworks"
$env:BINDGEN_EXTRA_CLANG_ARGS = "--target=x86_64-apple-darwin -isysroot ./macos-sysroot -F $FW -iframework $FW -I ./macos-sysroot/usr/include"
cargo zigbuild --release --target x86_64-apple-darwin   # Intel Mac
# Для Apple Silicon замените x86_64 на aarch64
cargo zigbuild --release --target aarch64-apple-darwin  # Apple Silicon
```

**Результаты:**
- `target/x86_64-apple-darwin/release/ter-music-rust` — Intel Mac
- `target/aarch64-apple-darwin/release/ter-music-rust` — Apple Silicon (M1/M2/M3/M4)

**Развёртывание на MacOS**:

```bash
# 1. Скопируйте на хост MacOS
scp ter-music-rust user@mac-host:~/

# 2. Сделайте исполняемым
chmod +x ter-music-rust

# 3. Разрешите запуск бинарного файла неизвестного источника
xattr -cr ter-music-rust

# 4. Запустите (дополнительные аудио-библиотеки не требуются)
./ter-music-rust -o /path/to/music
```

> Примечание: кросс-компиляция MacOS требует заголовков MacOS SDK; этот проект уже включает `macos-sysroot/`.
> Также требуется `libclang.dll` (установите через MSYS2 или LLVM).

## 🛠️ Структура проекта

```text
src/
├── main.rs       # Точка входа (разбор аргументов, инициализация, восстановление/сохранение конфигурации)
├── defs.rs       # Общие определения (перечисления PlayMode/PlayState, структуры MusicFile/Playlist)
├── audio.rs      # Управление аудио (обёртка rodio, воспроизведение/пауза/перемотка/громкость/прогресс)
├── analyzer.rs   # Анализатор аудио (громкость RMS в реальном времени, EMA-сглаживание, рендеринг волновой формы)
├── playlist.rs   # Управление плейлистом (сканирование каталога, параллельная загрузка длительности, выбор папки)
├── lyrics.rs     # Разбор текстов (LRC, локальный поиск, определение кодировки, фоновая загрузка)
├── search.rs     # Онлайн-поиск/загрузка (поиск Kuwo + Kugou + NetEase, загрузка, получение комментариев, потоковый запрос информации о песне)
├── config.rs     # Управление конфигурацией (сериализация JSON, 8 постоянных элементов)
└── ui.rs         # UI (рендеринг терминала, обработка событий, многовидовый режим, система тем/языков)
```

## 🐛 Устранение неполадок

### Ошибки сборки

```powershell
# Обновите Rust
rustup update

# Очистите и пересоберите
cargo clean
cargo build --release
```

### `link.exe не найден`

Установите Visual Studio Build Tools (см. Вариант 1 выше).

### `dlltool.exe не найден`

Установите полную инструментальную цепочку MinGW-w64 (см. Вариант 2 выше).

### Отсутствуют runtime DLL (инструментальная цепочка GNU)

```powershell
Copy-Item "C:\msys64\mingw64\bin\libgcc_s_seh-1.dll" -Destination ".\target\release\"
Copy-Item "C:\msys64\mingw64\bin\libstdc++-6.dll" -Destination ".\target\release\"
Copy-Item "C:\msys64\mingw64\bin\libwinpthread-1.dll" -Destination ".\target\release\"
```

### Аудиоустройство не найдено

1. Убедитесь, что аудиоустройство системы работает
2. Проверьте настройки громкости Windows
3. Попробуйте воспроизвести системный тестовый звук

### Проблемы рендеринга UI

- Убедитесь, что размер окна терминала не менее 80×25
- Для лучшего опыта используйте Windows Terminal
- В CMD убедитесь, что выбранный шрифт поддерживает CJK при необходимости

### Онлайн-поиск / загрузка текстов не удаётся

- Проверьте сетевое подключение
- Некоторые песни могут требовать VIP-доступ или быть удалены
- Файл текстов должен быть в допустимом стандартном формате LRC

### Запрос информации о песне не удаётся

- Когда API Key не настроен, автоматически используется бесплатная модель OpenRouter — ручная настройка не требуется
- Для использования пользовательской конечной точки нажмите `k` и введите базовый URL API, API Key и имя модели последовательно
- Поддерживается любой API, совместимый с OpenAI (DeepSeek, OpenRouter, AIHubMix и т.д.)
- Проверьте сетевое подключение к соответствующему API-сервису

### Медленная первая сборка

Первая сборка загружает и компилирует все зависимости; это ожидаемо. Последующие сборки будут значительно быстрее.
